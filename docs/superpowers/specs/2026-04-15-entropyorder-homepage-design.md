# 熵基秩序 EntropyOrder 主页设计文档

**日期：** 2026-04-15  
**版本：** v1.0  
**状态：** 已确认

---

## 1. 项目概述

为数据公司 **熵基秩序 EntropyOrder** 设计并实现一个单页 React 主页。整体风格为**玻璃科技风**，以**蓝白清新**为主轴，支持**明暗模式切换**与**中英双语切换**。页面需具备现代感、创新性，并包含适度的动画与 3D 交互元素。

### 核心需求
- 四个导航选项卡：**主页**、**产品**、**AI4SS4AI**、**BenchWithModel**（外部链接）
- 单页应用（SPA），点击导航平滑滚动到对应区块
- 响应式设计，全面适配移动端与桌面端
- 底部包含公司联系方式与咨询邮件表单
- 无后端，所有数据静态内嵌，邮件通过第三方服务（Formspree/EmailJS）发送

---

## 2. 设计原则与视觉风格

### 整体风格：熵域粒子（Entropy Field）
- **浅色模式：** 纯净白到淡蓝的柔和渐变背景，粒子为淡蓝色
- **深色模式：** 深蓝到近黑的过渡背景，粒子为亮蓝色和青色
- **玻璃态（Glassmorphism）：** 导航栏、卡片、浮层面板使用 `backdrop-blur` 和半透明边框
- **动效语言：** 粒子聚散象征“熵”与“秩序”的转化；3D艺术品体现数据的多维性与智能的涌现

### 色彩系统
| 元素 | 浅色模式 | 深色模式 |
|------|---------|---------|
| 主背景 | `#ffffff` → `#eff6ff` | `#0f172a` → `#1e293b` |
| 主色调 | `#2563eb` (blue-600) | `#3b82f6` (blue-500) |
| 强调色 | `#06b6d4` (cyan-500) | `#22d3ee` (cyan-400) |
| 文字主色 | `#1e293b` (slate-800) | `#f8fafc` (slate-50) |
| 文字次色 | `#64748b` (slate-500) | `#94a3b8` (slate-400) |
| 边框/分割线 | `rgba(37, 99, 235, 0.15)` | `rgba(59, 130, 246, 0.2)` |

## 3. 页面结构与导航

### 3.1 导航栏（Navbar）
- **位置：** 固定悬浮在页面顶部（`position: fixed`）
- **样式：** 毛玻璃效果，高度 64px，底部带 1px 半透明边框
- **左侧：** Logo 图片 + 公司名“熵基秩序 EntropyOrder”，点击平滑滚动回 Hero 顶部
- **中间/右侧：**
  - 导航项：主页、产品、AI4SS4AI、BenchWithModel
  - BenchWithModel 为新标签页打开的外部链接（占位链接 `#`）
  - 语言切换按钮（中 / EN）
  - 明暗模式切换按钮（太阳/月亮图标）
- **移动端：** 导航项折叠为汉堡菜单，点击后全屏/侧滑展开

### 3.2 页面区块顺序
1. **Hero** — 主页
2. **Products** — 产品
3. **AI4SS4AI** — 愿景
4. **Footer** — 联系方式与邮件表单

## 4. Hero 主页

### 4.1 布局与内容
- **全屏高度（100vh）**，Canvas 粒子网络作为背景层
- **中央内容区：**
  - Logo（大）
  - 双语 Slogan：**熵基秩序 · EntropyOrder**
  - 副标题（中英双语）：**构建高质量数据，驱动智能未来** / *Building High-Quality Data to Power the Future of Intelligence*
- **底部：** 一个向下滚动的微动画箭头（鼠标滚动提示）

### 4.2 粒子动画效果
- 使用 HTML5 Canvas 实现轻量粒子网络
- 粒子缓慢随机浮动，相邻粒子在距离较近时自动连线
- **鼠标交互：** 鼠标移入时，附近粒子向光标微微汇聚（“熵减聚变”），移开后自然散开
- 粒子数量与连线密度在移动端自动降低，保证性能

## 5. Products 产品页

### 5.1 整体布局
- 页面顶部一个小标题：**产品与服务** / *Products & Services*
- 分为三个大分类区块，每个区块采用**左右交替布局**：
  - 高水平专家数据集 → 图左文右
  - 多模态数据集 → 图右文左
  - Agent数据集 → 图左文右

### 5.2 分类导航
- **PC端：** 左侧固定粘性分类导航栏，点击后右侧平滑滚动到对应分类
- **移动端：** 顶部横向滑动的胶囊标签，点击切换/滚动定位

### 5.3 3D 交互式艺术品
每个分类配一个轻量级 React-Three-Fiber 抽象几何体，鼠标移上时有视角跟随和发光变化：

| 分类 | 3D艺术品描述 |
|------|-------------|
| 高水平专家数据集 | 缓慢自转的多面体晶体（Icosahedron），象征高阶智能。表面有淡蓝色流光。 |
| 多模态数据集 | 三条交织流动的光带（TorusKnot 或 Tube），分别代表视频、音频、文本三种模态。 |
| Agent数据集 | 在空间中自主漂浮的小球体集群（多个 Sphere），象征智能体的自主探索。 |

**性能降级策略：** 在移动端或低性能设备上，3D 艺术品自动降级为 CSS/SVG 动画版本。

### 5.4 数据集卡片
每个数据集以玻璃态卡片展示，包含：
- **名称**
- **简介**
- **数据规模**
- **标签**（如：中英 / 多模态 / Agent 等）
- **“联系获取样例”按钮**

**动效：**
- 卡片入场：错落的飞入动画（Framer Motion `staggerChildren`）
- Hover：3D 倾斜（`transform: perspective + rotateX/Y`）和边缘淡蓝发光

### 5.5 数据集内容

#### 高水平专家数据集
- HLE 数据集
- SFE 数据集
- MicroVQA 数据集
- 跨语言数据集

#### 多模态数据集
- 跨模态数据集（视频-音频-文本）

#### Agent数据集
- Browser 数据集
- Openclaw 数据集

### 5.6 “联系获取样例”交互
- 点击按钮弹出一个小型模态框（Modal）
- 表单字段：姓名、单位、邮箱、用途说明
- 提交后通过第三方邮件服务（Formspree）发送至公司邮箱
- 成功/失败均有提示

## 6. AI4SS4AI 页面

### 6.1 内容定位
采用**科技产品风格**，强调愿景、应用场景与未来价值，而非学术论文风格。

### 6.2 核心叙事
**以社会科学理解智能，以人格多样性驱动通用人工智能**

垂直时间轴展示三个节点：
1. **构建人格库** — 从社会科学理论出发，构建覆盖广泛的人格画像库
2. **生成多样数据** — 基于人格库驱动，生成具有高多样性、高代表性的数据
3. **助力 AGI** — 为通用人工智能的发展提供更丰富、更贴近人类真实的数据基础

### 6.3 视觉呈现
- 垂直时间轴，三个节点依次排开
- 每个节点配一个动态图标（CSS 动画或 Lottie）
- 节点之间的连线随滚动逐渐点亮（`useScroll` + `useTransform`）
- 节点周围穿插简短的中英双语说明文字
- 背景：淡淡的网格渐变，不抢夺主体注意力

## 10. 响应式与性能策略

### 10.1 断点
- 移动端：`< 768px`
- 平板：`768px - 1024px`
- 桌面端：`> 1024px`

### 10.2 性能优化
- **Canvas 粒子：** 移动端减少粒子数量（约 PC 端的 40%），降低连线检测频率
- **3D 降级：** 通过 `useMediaQuery` 检测移动设备或低性能模式，3D 组件替换为 CSS/SVG 动画
- **图片优化：** Logo 使用适当压缩的 PNG，必要时转为 WebP
- **动画节制：** 尊重 `prefers-reduced-motion`，为对动画敏感的用户提供减弱动画的选项

## 11. 明暗模式与国际化

### 11.1 主题切换
- 使用 Tailwind 的 `dark` 类策略
- 主题状态保存在 `localStorage`，下次访问自动恢复
- 默认跟随系统偏好 `prefers-color-scheme`

### 11.2 语言切换
- 支持中文（zh）和英文（en）
- 切换按钮在导航栏右侧
- 语言偏好保存在 `localStorage`
- 所有可见文本均通过 i18n 键管理，不允许硬编码

---

## 12. 邮件表单策略

### 12.1 第三方服务
- 使用 **Formspree** 的免费方案（无需后端，支持 AJAX 提交）
- 表单 `action` 指向唯一的 Formspree endpoint
- 提交字段包含：`name`, `organization`, `email`, `purpose`（产品页样例申请）或 `name`, `email`, `subject`, `message`（页脚咨询）

### 12.2 交互反馈
- 提交中：按钮变为 Loading 状态
- 提交成功：显示绿色勾选动画 + 成功提示文字
- 提交失败：显示红色错误提示，保留用户已填写内容
