*我们如何通过人机协作架构，制造最强模型也无法回答的问题*

---

2025 年初，一项名为 Humanity's Last Exam（HLE）的 benchmark 在学术界引起了广泛关注。来自全球近五百个机构的领域学者，跨越数学、物理、生物、法律、语言学等数十个学科，联合出具了约 3000 道专家级题目，目的是测试当前最强 AI 模型的知识极限。结果令人清醒：GPT-4o 的准确率为 3.3%，Claude 3.5 Sonnet 为 4.3%，即便是推理能力最强的 Gemini Thinking 也仅有 6.2%¹。这些数字说明了一个迟早要被正视的事实——当前沿模型在常规 benchmark 上趋近饱和时，**真正的能力瓶颈不再是架构和算力，而是数据**。更准确地说，是那些足以挑战模型知识边界的、专家级的训练数据。

我们的工作正是围绕这个问题展开。这篇文章不是一份产品说明书，而是一次对核心技术路线的公开思考：如何系统化地、可规模化地制造这类数据？我们为什么认为这个问题的本质不是"标注"，而是"知识工程"？

---

## 一个诱人但走不通的直觉方案

面对专家级数据的需求，第一个自然的想法是：用 GPT-4 或 Claude 这样的前沿模型直接从学术文献中生成难题。这条路看起来既经济又能 scale——但我们在实践中发现，它存在四个结构性障碍。这些障碍不是 prompt 写得不好能解决的，它们是当前范式的内生局限。

第一个障碍是**自举困境**。模型不可能稳定地生成超出自身推理能力上限的问题。这是一个逻辑约束——如果模型能生成一道题并正确验证答案，那这道题大概率无法对同等能力的模型构成有效挑战。用一个不够严谨但足够直觉的类比：你无法通过左脚踩右脚让自己离开地面。Khan 等人（2024）对 scalable oversight 的研究也佐证了这一点：当弱模型试图监督强模型、甚至同等能力的模型互相评判时，存在不可消除的系统性盲区²。

第二个障碍在于**边际领域知识的缺失**。一道真正的专家级难题，几乎从不只依赖论文中写明的显式信息。它还依赖领域内的隐性知识（tacit knowledge）——那些"材料科学的博士都知道但论文里不会写"的实验直觉、方法论常识和反直觉经验。HLE 之所以难，原因之一正是每道题由活跃的领域研究者出具，编码了只有从业者才拥有的边际知识。这类知识不在 LLM 的训练分布中，无法通过 prompt engineering 召回。

第三个障碍来自**幻觉与可审计性的缺失**。在无约束生成中，LLM 无法区分"从训练数据中检索到的事实"和"在概率上看起来合理但实际编造的内容"。在通用对话场景中这可以被容忍，但在专家级数据生产中，每一条数据都必须可追溯到明确的源文证据，否则无法通过质量审核。Bavaresco 等人（2024）的研究表明，LLM-as-a-Judge 在专家知识任务上与领域专家的评估一致率仅为 64-68%³，而 Li 等人（2025）进一步指出，缺乏人类锚点的 LLM 自动评估从本质上不可信赖⁴。

第四个障碍是**推理深度的天花板**。让 LLM 直接从文本生成问题时，模型有强烈的倾向生成单跳（1-hop）的事实回忆型问题，例如"本文使用了哪个数据集？"或"该方法在哪个指标上取得了最好结果？"。但真正具有区分度的专家级题目需要跨概念、跨章节、多步骤的整合推理——而这恰恰是 LLM 自由生成时最不擅长的。AutoQG 的实验提供了量化数据支撑：当推理路径从 1-hop 增加到 3-hop 时，base model 的准确率从 88.5% 下降到 67.9%，即便是 reasoning model 也从 89.3% 降至 81.2%⁵。推理深度是难度最有效的单一杠杆，但 LLM 自身并不善于主动构造长推理链路。

这四个困境互相强化，共同指向一个结论：纯自动化的数据生产管线存在结构性上限。突破它需要一个不同的架构范式。

---

## 我们的技术路线：为每个困境设计对应的架构

如果你接受上述诊断，那解法的逻辑就相当自然了。每个困境指向一个需要填补的能力缺口，而我们的架构设计就是围绕这些缺口展开的。

打破自举困境和补充边际知识的方法只有一个——引入外部知识源，具体来说是领域专家。但关键在于：专家在我们的框架中不是标注执行者，而是**知识架构师**。他们的核心工作是设计领域本体（ontology）——定义在这个学科中什么实体类型是重要的、什么关系类型是有意义的、什么粒度是合适的。van Cauter 和 Yakovets（2024）在工业维修领域的研究给了我们一个关键的实证信号：仅需 20 条专家标注的种子样本，配合一份领域 ontology 定义，就能让 Llama-3-70B 的实体-关系提取达到 F1 0.69，接近需要完整微调的传统模型（F1 0.77）⁶。20 条标注——不是 2000 条，不是 20000 条。这意味着**专家的投入被极大地杠杆化了**，他们的知识以 ontology 的形式编码进系统，然后被 LLM 的生成能力上千倍地放大。

解决幻觉和推理深度问题的方法，则是在 LLM 的输出和最终数据之间插入一个**结构化的中间层——知识图谱（Knowledge Graph）**。这个选择需要解释一下。我们不直接从原始文本生成题目。整个流程是先将文献文本转化为由实体-关系三元组构成的结构化网络，其中每条边都绑定到原文的具体章节和文本片段——这就彻底解决了溯源问题。然后，无法通过 ontology schema 验证或无法在原文中找到锚点的三元组会被系统识别并剔除——这是对幻觉的结构性约束，它不依赖于对 LLM 输出的概率判断，而是依赖于逻辑层面的可验证性。

同时，知识图谱天然提供了 multi-hop 推理的脚手架。一条从 entity₁ 经过 relation₁ 到达 entity₂，再经过 relation₂ 到达 entity₃ 的路径，本身就是一条推理链。基于这些路径——而非原始文本——生成问题，模型被迫遵循显式的推理结构，而路径的 hop 数直接对应问题的推理深度。由此，难度变成了一个可被精确控制的工程参数。

Bian（2025）对 LLM 驱动知识图谱构建的综合调研中提出了一个我们深表认同的判断：当前 LLM×KG 的技术演化正在从静态 schema 转向动态归纳，从模块化流水线转向生成式统一框架——但 schema-based 路线仍然是高质量数据生产的必要条件⁷。自由生成可以带来覆盖率，而唯有结构化约束才能带来可靠性。

在提取环节有一个重要的设计选择值得展开。科学文献不是扁平的文本流。Abstract 提出主张，Methods 描述过程与配置，Results 呈现证据，Discussion 进行归因和展望。同一篇论文的不同 section 承载着不同的话语功能，对它们施加相同的提取策略是一种信息损失。因此我们采用章节感知（section-aware）的提取策略——针对不同 section 设计不同的提示模板和 ontology 约束。举一个具体的对比：对于 Methods 部分的一句话"We trained Model-A on the CIFAR-100 dataset with a learning rate of 0.001"，不做章节区分的 flat prompting 通常只会提取出 (Model-A, trainedOn, CIFAR-100)，而章节感知提取能够产出 (Model-A, evaluatedOn, CIFAR-100) 和 (Model-A, hasHyperparameter, learning rate=0.001) 两条更精确、信息密度更高的三元组。差别的根源在于：提取策略编码了"在 Methods 部分，超参数配置是有意义信息"这条领域先验。

## 不是质检，是质量工程

一条三元组通过了事实性验证不代表整张知识图谱是合格的。一个由正确的局部片段组成的整体仍可能是有缺陷的——实体粒度在不同章节间不一致、跨节信息存在隐含矛盾、关键关系被遗漏。我们需要的是图级别的系统审计，而非逐条检查。

为此，我们构建了一套多维评估框架。给定候选知识图谱 $G$，评估模块从五个正交维度进行审计，产出一个质量向量：

$$\mathbf{q}(G) = (s_{\text{dom}},\ s_{\text{acc}},\ s_{\text{con}},\ s_{\text{com}},\ s_{\text{gra}}) \in [0, 1]^5$$

其中 $s_{\text{dom}}$ 衡量提取结果与目标学科的契合程度，$s_{\text{acc}}$ 衡量三元组的事实正确性，$s_{\text{con}}$ 检测跨章节的矛盾与冲突，$s_{\text{com}}$ 评估关键关系的覆盖率，$s_{\text{gra}}$ 检查实体和关系的抽象层级是否恰当。整体图质量由加权聚合给出：

$$Q(G) = \sum_{i=1}^{5} w_i \cdot s_i, \quad \sum_{i=1}^{5} w_i = 1$$

权重 $\{w_i\}$ 并非均匀分配，而是通过消融实验校准的——我们发现准确性和一致性对最终数据质量的预测力最强，因此在评估中被赋予更高权重。

这套评估不是一次性打分然后通过或不通过。它的核心设计是一个迭代式的自校正回路。当 $Q(G) < \tau$ 时，评估模块不只输出分数，还生成结构化的诊断反馈——例如"Methods 部分出现高密度低置信度三元组"或"实体 X 在 Methods 和 Results 中存在不一致引用"——这些反馈被直接注入下一轮提取的上下文中，引导生成器进行定向修正：

$$G^{(t+1)} = \mathcal{R}\big(G^{(t)},\ \mathbf{q}(G^{(t)})\big)$$

迭代持续直至 $Q(G) \geq \tau$，最终产出通过审计的知识图谱 $G^{\star}$。

但我们不会声称这套系统完全不需要人。恰恰相反，有两个环节我们认为必须保留人的判断。第一是评估维度的定义与权重校准——"什么叫粒度合适"在 NLP 领域和生物医学领域是完全不同的标准，这是领域判断而不是技术参数。第二是边界案例的仲裁——当 `basedOn` 和 `uses` 难以区分，当两个章节的信息看似矛盾实际上是不同实验条件下的合理差异时，自动化系统会标记不确定性，而最终的拍板属于领域专家。

## 从推理路径到专家级难题

知识图谱 $G^{\star}$ 本身不是产品，它是基础设施。最终产品是从中生成的、具有精确推理深度和证据可追溯性的专家级问题。

生成过程从推理路径的识别开始。我们在 $G^{\star}$ 中搜索满足特定约束的 multi-hop 路径——非环路、跨章节、语义连贯——然后将路径和对应的原文证据一起输入 schema-constrained 的生成模块。这里有一个重要的设计意图：路径遍历不是随机游走，而是结构约束的。跨章节路径（例如从 Methods 中的模型配置到 Results 中的性能指标再到 Discussion 中的因果归因）天然要求信息整合，比 section 内部的路径具有更高的认知难度。

hop 数量是难度控制最直接的杠杆。这不只是我们的假设——AutoQG 在 4000 余篇论文上的大规模实验给出了清晰的量化证据：对于 base model，1-hop 题目准确率约 88%，2-hop 降至 81%，3-hop 进一步降至 68%。reasoning model 在各级上维持更高水平但同样呈现单调下降⁵。这意味着推理深度作为难度 proxy 具有很好的单调性和区分度。

但推理链路的长度并不是难度的全部。一道真正的专家级选择题，它的核心壁垒不在于正确答案有多隐蔽，而在于干扰项有多"像对的"。设计具有高度迷惑性的干扰选项——那些"看起来完全合理，但只有真正理解该实验设置的人才能排除"的选项——这件事目前仍然高度依赖人类的领域判断力。AI 可以批量生成候选干扰项，但判断一个干扰项是否对该领域的博士生具有迷惑力，这属于人的独占能力。

## 越做越深的领域壁垒

我们在多个学科领域的实践中积累了一些观察。这些观察不涉及具体的客户项目，但它们是理解这个行业竞争格局的关键。

不同领域的知识图谱提取，其典型错误模式具有高度的领域特异性。在方法论密集的工程类文献中，最常见的错误是关系方向反转——"A 基于 B"被提取为"A 使用 B"，表层的动词模式（"use", "incorporate"）误导了关系分类。在实验科学领域，因果性过度推断更为频发——"A 与 B 相关"被升级为"A 导致 B"。在概念层级丰富的领域（如生物分类学或知识体系研究），粒度失配是主要问题——一个应当被表达为单一抽象关系的语义被拆分成了多条过于细碎的三元组⁸。这些错误不是通过更好的通用 prompt 能解决的，因为它们的根源在于领域知识结构本身。我们为每个深度参与的领域建立了专属的错误模式库和对应的修正策略——这是一种只能通过实践积累、无法从论文中学到的 know-how。

Ontology 也不是一次性设计好就永远适用的。领域在持续演化——新的概念被引入、旧的术语被重新定义、研究范式在迁移。一个 2023 年为 NLP 领域设计的 ontology，到了 2025 年可能无法充分覆盖 agent 和 tool-use 范式下涌现的新关系类型。我们持续维护和更新多个领域的 ontology，每次更新本身就是一次对领域的深度重新理解。

这种积累构成了一个数据飞轮（data flywheel）⁹。每一次项目交付都沉淀为更完善的领域 schema、更精确的质检规则、更丰富的种子样本库。这使得下一次同领域的生产能在更高的起点上启动，单位成本更低，质量天花板更高。对于新进入者来说，这不仅仅是技术差距——而是需要时间和项目才能弥合的经验差距。

## The Bottleneck Has Shifted

HLE 之后，行业需要直面一个现实：模型能力的下一个增量，越来越难从架构创新和规模扩展中获得，而更可能来自训练数据质量的提升——具体来说，是来自那些当前模型自己无法生成、无法验证、无法自我突破的数据。

与此同时，知识图谱正从数据生产流程中的一个工程组件，演化为大语言模型的认知基础设施。RAG 系统依赖结构化知识做事实锚定，Agent 框架依赖它做推理支撑和记忆管理¹⁰。上游知识图谱的质量，正在成为这些下游系统可靠性的上限。

而数据生产的本质，从未因 AI 的进步而改变——它始终是**将人类的知识以机器可用的形式编码**。变化的是编码的方式：从手工逐条标注，到以 ontology 为载体的知识工程；从按人头计价的劳动密集模式，到以领域理解为核心资产的知识密集模式。

这不是 AI 取代人。这是 AI 放大了专家知识的价值。而模型的能力边界——最终取决于它被喂养的知识的质量。
